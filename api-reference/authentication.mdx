## Overview

The Dromo API uses API keys to authenticate requests. You can view and manage your API keys in the [Dromo Dashboard](https://dashboard.dromo.io).

Your API keys carry many privileges, so be sure to keep them secure! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.

## Authentication Methods

<Tabs>
  <Tab title="Bearer Token">
    Include your API key in the Authorization header:

    <CodeGroup>
    ```bash cURL
    curl -H "Authorization: Bearer YOUR_API_KEY" \
         https://app.dromo.io/api/v1/schemas
    ```

    ```javascript JavaScript
    const response = await fetch('https://app.dromo.io/api/v1/schemas', {
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
      }
    });
    ```

    ```python Python
    import requests

    headers = {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    }

    response = requests.get('https://app.dromo.io/api/v1/schemas', headers=headers)
    ```
    </CodeGroup>
  </Tab>

  <Tab title="Header Key">
    Alternatively, use the custom header:

    <CodeGroup>
    ```bash cURL
    curl -H "X-Dromo-License-Key: YOUR_API_KEY" \
         https://app.dromo.io/api/v1/schemas
    ```

    ```javascript JavaScript
    const response = await fetch('https://app.dromo.io/api/v1/schemas', {
      headers: {
        'X-Dromo-License-Key': 'YOUR_API_KEY',
        'Content-Type': 'application/json'
      }
    });
    ```
    </CodeGroup>
  </Tab>
</Tabs>

## Getting Your API Key

<Steps>
  <Step title="Sign in to Dashboard">
    Navigate to the [Dromo Dashboard](https://dashboard.dromo.io) and sign in to your account.
  </Step>
  <Step title="Access API Settings">
    Go to **Settings** â†’ **API Keys** in the navigation menu.
  </Step>
  <Step title="Generate Key">
    Click **Generate New API Key** and copy the key to a secure location.
  </Step>
  <Step title="Test Authentication">
    Use the key to make a test API call and verify it works correctly.
  </Step>
</Steps>

## API Key Management

<Warning>
  **Keep your API keys secure!** 
  - Never expose them in client-side code
  - Store them securely in environment variables
  - Rotate keys regularly for enhanced security
  - Delete unused keys immediately
</Warning>

### Key Permissions

<ResponseField name="read" type="boolean">
  Allows reading schemas, imports, and user data
</ResponseField>

<ResponseField name="write" type="boolean">
  Allows creating and updating schemas and configurations
</ResponseField>

<ResponseField name="delete" type="boolean">
  Allows deleting schemas and import data
</ResponseField>

## Error Handling

When authentication fails, you'll receive a `401 Unauthorized` response:

<CodeGroup>
```json Example Error Response
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid API key provided",
    "details": "The API key is either missing, invalid, or has been revoked"
  }
}
```
</CodeGroup>

<Info>
  **Rate Limiting**: API keys are subject to rate limiting. See our [rate limiting guide](/api-reference/rate-limits) for details.
</Info>
